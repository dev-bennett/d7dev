1. Search knowledge/ for existing articles on this topic
2. Determine the appropriate location:
   - knowledge/domains/<domain>/ -- domain-specific knowledge
   - knowledge/data-dictionary/ -- metric/field definitions
   - knowledge/runbooks/ -- operational procedures
   - knowledge/decisions/ -- analytical decision records (ADR format)
3. If updating existing article:
   - Show current content and proposed changes
   - Require user confirmation before modifying
   - Add "Last updated: <date>" and change summary
4. If creating new article:
   - Use appropriate template/format for the article type
   - Domain articles: context, key metrics, stakeholders, related analyses
   - Data dictionary: field name, definition, calculation, source, owner
   - Runbooks: purpose, steps, prerequisites, troubleshooting
   - Decisions: context, decision, rationale, consequences, date
5. Cross-reference: check if the update affects other KB articles
6. Stage modified/created files

Knowledge base articles should be authoritative and concise.
Always cite the source of information (analysis, stakeholder, documentation).
