# Inventory — Runtime Substrate Audit

@../CLAUDE.md

## Purpose
Raw Phase-1 enumeration and Phase-2 interaction mapping for Epic 0.1. Each file here captures the direct observation of one or more runtime resources; the final synthesis ships to `knowledge/runbooks/runtime-substrate-catalog.md`.

## Conventions
- One resource class per file (memory.md, transcripts.md, settings.md, etc.) OR one consolidated enumeration file, per what the evidence supports.
- Every claim cites an observed path or command output. No narration without evidence.
- Lifecycle cues (mtime, rotation suffixes, gitignored vs tracked) are captured at observation time and preserved.
- Output format: plain markdown headings + tables. No rhetorical prose.
- Files here are scratch working artifacts — do not cross-reference them from stakeholder artifacts once the runbook ships.
